# 쇠사슬 UI/UX 가이드라인 — 모바일 · 바이럴 중심

> **North Star:** 수익이 아니라 **확산**이 목적. 그래서 모든 화면의 기준은 하나 —
> **"이 순간을 캡처해서 스토리에 올리고 싶은가? 이 링크를 친구에게 보내고 싶은가?"**
> UI가 예쁜지가 아니라, **UI가 사람을 데려오는가**로 판단한다.

---

## 0. 3-10-1 법칙 (첫인상)
- **3초**: 열자마자 "이게 뭐 하는 앱"이 사슬 히어로 하나로 전달돼야 함
- **10초**: 첫 인증까지 마찰 0 (부위 고민 X, 사진 선택)
- **1탭**: 자랑/공유가 항상 한 탭 거리

가입 전 이탈이 가장 큰 손실. **초대 링크는 가입창이 아니라 "살아있는 크루 + 내 자리"부터 보여준다(빈 고리).**

---

## 1. 모바일 UX 원칙

**① 엄지 존(Thumb Zone) 우선**
- 핵심 액션(인증/운동 시작/응원)은 **항상 화면 하단 고정 바**. (이미 적용)
- 상단은 정보(사슬·상태), 하단은 행동. 위로 뻗지 않게.

**② 마찰 제로 인증**
- 인증 = 종목 탭 1번 + (선택)사진 = 끝. **필수 입력 최소화.**
- 사진은 **카메라 바로 실행**(`capture`), 갤러리 선택도 1탭.
- 텍스트 입력은 전부 선택. 강제하는 순간 이탈.

**③ 즉각적 보상 (도파민 루프)**
- 인증 = 고리가 **철컹 튕겨 붙고**, 숫자 카운트업, "+1" 버스트. (이미 적용)
- 등급업·주간 완성은 더 크게 축하(색종이/풀스크린 반짝).
- **행동 → 0.3초 안에 시각 보상.** 지연은 습관을 못 만든다.

**④ 한 화면 = 한 목적**
- 스크롤은 위→아래로 이야기 흐르게: 사슬(자부심) → 이번주(긴장) → 갤러리(자랑) → 보드(관계).
- 모달은 바텀시트로(엄지로 닫기 쉽게). (이미 적용)

**⑤ 로딩·빈 상태도 설계**
- 빈 크루: "첫 고리를 채워보세요" + 큰 CTA. 절대 텅 빈 흰 화면 금지.
- 로딩은 스켈레톤/사슬 애니메이션으로 브랜드 각인.

## 2. 비주얼 시스템

| 요소 | 규칙 |
|---|---|
| **주인공** | 사슬(고리). 크루 화면 첫 스크롤은 무조건 사슬 히어로 |
| **컬러** | 밝은 베이스(#f2f4f7) + 오렌지(행동 #f5501a) + 골드(사슬 #e39a00). 3색 이상 남발 금지 |
| **타이포** | 숫자(고리 수)는 크고 굵게 — 성취가 곧 타이포. 본문은 담백하게 |
| **여백** | 답답함이 이탈. 카드 사이·안쪽 여백 넉넉히, 부드러운 그림자로 떠 보이게 (적용됨) |
| **모션** | 모든 상태 변화에 0.2~0.5초 이징. 단, 화려함≠과함. 보상 순간에만 크게 |
| **아이콘** | 종목은 이모지로 즉시 인지(🏋️🏃🏊🧗). 텍스트보다 아이콘 먼저 |
| **라운드** | 큼직한 라운드(16~24px) = 친근·젊은 느낌. 대학생 타깃에 맞음 |

**감정 곡선:** 사슬(뿌듯) → 미달 경고(긴장, 붉은 톤) → 인증 보상(쾌감) → 갤러리(자랑). 화면이 감정을 태워야 다시 연다.

## 3. 바이럴 설계 (핵심 — 여기가 승부처)

### 3-1. 공유 카드 (최우선 무기)
성취 순간마다 **스토리 규격(1080×1920, 9:16) 이미지 자동 생성** → 원탭 공유.
- **담을 것:** 사슬 비주얼 + `N고리` 대문짝 + 등급 뱃지 + 크루명 + 인증샷 콜라주(2~4장) + 하단에 **로고 + `soesaseul` URL/코드**
- **디자인:** 배경은 브랜드 그라데이션, 중앙에 사슬, 숫자는 초대형. 스토리에 올렸을 때 **3m 밖에서도 "N고리"가 읽혀야** 함
- **왜:** 보상이 곧 광고. 친구가 "이거 뭐야?" → 링크 → 유입. 무자본 획득의 엔진

### 3-2. 공유가 터지는 순간(트리거) — 그때 공유 버튼을 들이민다
| 순간 | 카드 문구 |
|---|---|
| 등급업 | "🔥 우리 크루, 불사슬 달성!" |
| 시즌 완주 | "8주 완주 ⛓️ 우리 47고리" |
| 최고기록 갱신 | "역대 최고! 52고리" |
| 주간 정산(일요일) | "이번 주 우리 크루 리포트" |
| 첫 인증 | "오늘 첫 고리 채웠다 💪" |

각 순간의 완료 화면에 **"📸 스토리에 자랑하기"**를 1순위 버튼으로.

### 3-3. 초대 링크 = 랜딩페이지 (가입 전 훅)
- `?code=` 링크는 열자마자 **살아있는 사슬 + 내 이름표가 붙은 빈 고리** (빈 고리 아이디어).
- **OG 이미지**(카톡/디스코드 미리보기)도 사슬 카드로 → 링크 자체가 광고 썸네일.
- 문구: "민준이 너를 지목했어. 네가 들어와야 이 사슬이 이어져."

### 3-4. 브랜드 각인
- 모든 공유 이미지 우하단에 **⛓️ 쇠사슬 + soesaseul URL** 워터마크. 스크린샷이 재유통돼도 출처가 따라감.
- 캡처하고 싶게 만든 뒤, 캡처에 브랜드를 심는다.

### 3-5. 자랑 유발(Vanity) 설계
- 등급/최고기록/총 인증 = **보여주고 싶은 숫자**로 크게.
- 프로필/공유 카드에 성취를 전시. "나 이렇게 꾸준해"를 증명하고 싶게.

---

## 4. 스토리/인스타 특화 규격
- **9:16 세로**, 상하 각 250px는 UI 가림 대비 **안전 영역** 비워두기
- 인스타 **링크 스티커** 유도(스와이프업 대체) — 카드에 "프로필 링크 / 코드 XXXX" 병기
- 공유는 **네이티브 공유시트**(`navigator.share`) 우선 → 인스타/카톡/디스코드 자동 노출
- 카드 텍스트는 **큰 숫자 + 짧은 한 줄**. 문장 길면 스토리에서 안 읽힘

---

## 5. 마찰 제거 체크리스트 (구현 기준)
- [ ] 인증에 필수 텍스트 입력이 있는가? → **없애라**
- [ ] 첫 화면에서 가치 전달까지 스크롤이 필요한가? → 접힘선 위로 끌어올려라
- [ ] 공유가 2탭 이상인가? → 1탭으로
- [ ] 초대받은 사람이 가입 전에 크루를 보는가? → 봐야 한다
- [ ] 성취 순간에 공유 버튼이 있는가? → 모든 완료 화면에
- [ ] 로딩/빈 상태가 흰 화면인가? → 스켈레톤·안내로

---

## 6. 안티패턴 (하지 말 것)
- ❌ 가입부터 시키기 (가치 먼저, 가입 나중)
- ❌ 인증에 폼 길게 받기 (탭 몇 번으로 끝)
- ❌ 공유 카드 없이 텍스트만 공유 (이미지가 퍼진다)
- ❌ 다크·저대비·정보 과밀 (대학생은 0.5초 안에 판단)
- ❌ 성취를 작게 표시 (숫자는 크게, 자랑거리로)
- ❌ 알림 남발 (미달 임박·응원 등 감정적으로 의미있을 때만)

---

## 7. 지금 바로 적용할 우선순위 (ROI 순)
1. **공유 카드 + "스토리에 자랑하기"** — 완료 화면(등급업·인증)마다. **바이럴 1순위**
2. **초대 링크 랜딩 = 빈 고리** — 가입 전 크루를 보여주는 훅
3. **OG 이미지**(링크 미리보기를 사슬 카드로) — 카톡/디스코드 확산
4. **인증 보상 강화**(등급업 풀스크린 축하) — 도파민 → 재방문
5. **빈 상태·로딩 브랜딩** — 이탈 방지

> 요약: **캡처하고 싶은 순간을 최대한 많이 만들고, 그 캡처에 브랜드와 링크를 심어라.** 이게 쇠사슬 UI/UX의 존재 이유다.
